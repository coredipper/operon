¥subsection{Compiler Functors}

A ¥textbf{compiler functor} $F: ¥mathrm{Arch}(¥text{Source}) ¥to ¥mathrm{Arch}(¥text{Target})$ maps one architecture to another. In our implementation, each compiler produces a serializable configuration dict for a specific orchestration framework:

¥begin{itemize}[leftmargin=*]
¥item ¥texttt{organism¥_to¥_swarms}: Operon $¥to$ Swarms graph workflow
¥item ¥texttt{organism¥_to¥_deerflow}: Operon $¥to$ DeerFlow LangGraph session
¥item ¥texttt{organism¥_to¥_ralph}: Operon $¥to$ Ralph hat configuration
¥item ¥texttt{organism¥_to¥_scion}: Operon $¥to$ Scion grove deployment
¥item ¥texttt{organism¥_to¥_langgraph}: Operon $¥to$ LangGraph StateGraph
¥end{itemize}

Each compiler is wrapped in a ¥texttt{CompilerFunctor} class that automatically extracts source and target architectures, verifies the functor laws, and produces a ¥texttt{PreservationResult} with three checks:

¥begin{enumerate}[leftmargin=*]
¥item ¥textbf{Graph preservation}: source stage names $¥subseteq$ target stage names, source edges $¥subseteq$ target edges.
¥item ¥textbf{Certificate preservation} (Prop‾5.1): source certificate theorems $¥subseteq$ target certificate theorems, and all certificates ¥texttt{verify()} $¥to$ ¥texttt{holds=True}.
¥item ¥textbf{Interface preservation}: source stage names present in target (mode mapping may differ).
¥end{enumerate}

¥subsection{The LangGraph Wrapper}

The LangGraph functor is special. ¥texttt{organism¥_to¥_langgraph()} wraps ¥texttt{organism.run()} as a ¥emph{single LangGraph node}. The organism's stages, components, and guarantees execute inside the node---LangGraph provides the execution host, Operon provides the structural control flow.

This makes the LangGraph functor a ¥textbf{certificate-preserving wrapper}: the compiled graph collapses the multi-stage topology into a single node, so graph structure ($G$) and interface ($¥Phi$) are not preserved in the target representation. However, all certificates in $¥mathrm{Know}$ are preserved because they are evaluated inside the wrapped ¥texttt{run()} call---the organism's own run loop enforces every structural guarantee internally.

¥begin{proposition}[Certificate-Only Preservation]
The LangGraph functor preserves $¥mathrm{Know}$ completely: for any organism $A$, all certificates attached to $A$ verify successfully after compilation. Graph topology and interface are collapsed into the single-node wrapper but remain enforced internally by ¥texttt{organism.run()}.
¥end{proposition}

The operational meaning: wrapping a harness in a different execution framework transfers all structural guarantees because the guarantees reside in $¥mathrm{Know}$, not in $G$ or $¥Phi$. The certificates hold because they depend on harness logic, not graph shape.
